### Dockerfile for Master Node (`masters/Dockerfile`)

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# System deps
RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# Python deps
COPY pyproject.toml README.md ./
RUN pip install --upgrade pip && \
    pip install -e .

# Copy source
COPY src/ ./src/

EXPOSE 8080 8082

CMD ["uvicorn", "nisha_master.main:app", "--host", "0.0.0.0", "--port", "8080"]
```

### docker-compose snippet

```yaml
  master:
    build: ./masters
    ports:
      - "8080:8080"   # Dashboard
      - "8082:8082"   # Agent WebSocket
    environment:
      - BACKEND_WS_URL=ws://backend:8000/api/v1/ws/realtime
      - MASTER_ID=MASTER_001
      - BUFFER_RAM_LIMIT_MB=1024
    depends_on:
      - backend
      - redis
    volumes:
      - ./masters/src:/app/src
      - /tmp/nisha_buffer:/tmp/nisha_buffer  # Persist buffer across restarts
```

### Systemd Service (production)

`/etc/systemd/system/nisha-master.service`

```ini
[Unit]
Description=NISHA Master Node
After=network.target postgresql.service redis.service
Wants=postgresql.service redis.service

[Service]
Type=simple
User=nisha
WorkingDirectory=/opt/nisha/masters
Environment="DATABASE_URL=postgresql://nisha:secret@localhost:5432/nisha_sentinel"
Environment="REDIS_URL=redis://localhost:6379/0"
ExecStart=/opt/nisha/masters/.venv/bin/uvicorn nisha_master.main:app --host 0.0.0.0 --port 8080
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### Key Architecture Points

- **Ports**: 8080 (dashboard HTTP), 8082 (agent WebSocket)
- **Framework**: FastAPI (same as backend, but smaller scope)
- **Queues**: `stream_queue` (5000), `telemetry_queue` (1000)
- **Buffer**: 1024 MB RAM + disk spill to `/tmp/nisha_buffer`
- **Backend Link**: WSS client with auto-reconnect (exponential backoff 1s→60s)
- **Agent Protocol**: Binary NISHA frames (24-byte header)
- **Dashboard**: FastAPI serving static HTML/JS, WebSocket push every 100ms
